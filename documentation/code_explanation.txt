# Code Documentation & Execution Sequence

This document explains the purpose of each script in the research pipeline and lists the current execution order used to reproduce the analysis.

---

## Stage 1: Data Acquisition
These scripts pull raw data from WRDS/SEC sources or prepare large reference files for the downstream pipeline.

1. **`01_data_acquisition/fetch_stock_prices.py`**
   - **Purpose**: Connects to WRDS/CRSP and downloads a broad monthly stock universe from 2008-01 to 2022-12 by default, including prices (`prc`), returns (`ret`), shares outstanding, volume, and identifiers. It also computes `market_cap`. Common-stock screening is intentionally deferred to the cleaning stage.
   - **Output**: Individual raw stock CSVs in `raw_data/stock_price/`.

2. **`01_data_acquisition/fetch_cik_mapping.py`**
   - **Purpose**: Builds the cross-database identifier bridge between SEC CIK, Compustat GVKEY, and CRSP PERMNO/Ticker.
   - **Output**: `raw_data/cik_ticker_permno_mapping.csv`.

3. **`01_data_acquisition/fetch_sp500_constituents.py`**
   - **Purpose**: Downloads annual end-of-year S&P 500 constituent snapshots used to define ex-ante large-cap universes for the regression windows. It now includes the 2007 year-end snapshot by default.
   - **Output**: `raw_data/sp500_annual_constituents_2007_2022.csv`.

4. **`01_data_acquisition/filter_lm_header.py`**
   - **Purpose**: Processes the large Loughran-McDonald 10-K header dataset and keeps only the geographic and filing-ID columns needed for headquarters assignment.
   - **Output**: `cleaned_data/LM_10K_Headers_Geo_Filtered.csv`.

5. **`01_data_acquisition/filter_mapping.py`**
   - **Purpose**: Optional refinement step for the identifier bridge before merging filings with securities.
   - **Output**: Refined CIK/GVKEY/PERMNO mapping file for downstream merges.

---

## Stage 2: Data Cleaning & Processing
These scripts clean the stock panel, map firms to MSAs, build benchmark portfolios, and inject the leave-one-out regressors used in the paper.

6. **`02_cleaning_processing/clean_stock_prices.py`**
   - **Purpose**: Performs primary stock-data cleaning. It first keeps common shares only (`SHRCD` 10 or 11), then drops invalid return/price rows, removes non-operating entities such as REITs and closed-end funds, standardizes `market_cap` into USD, and maps SIC codes into Fama-French 48 industries.
   - **Output**: Cleaned stock CSVs in `cleaned_data/stock_price/`.

7. **`02_cleaning_processing/merge_geo_stock.py`**
   - **Purpose**: Merges 10-K filing geography with the security mapping table, keeping only filing-security matches that are valid on the filing date.
   - **Output**: `cleaned_data/merged_geo_ticker_data.csv`.

8. **`02_cleaning_processing/build_monthly_zip_panel.py`**
   - **Purpose**: Converts the filing-level geography into a monthly ZIP-code panel, forward-filling the most recent headquarters ZIP for each ticker.
   - **Output**: `cleaned_data/monthly_zip_panel_180mo.csv`.

9. **`02_cleaning_processing/clean_zip_panel.py`**
   - **Purpose**: Cleans ZIP values in the monthly panel by trimming whitespace, keeping the first 5 digits, dropping all-zero values, and removing malformed ZIPs.
   - **Output**: `cleaned_data/monthly_zip_panel_180mo_cleaned.csv`.

10. **`02_cleaning_processing/build_msa_panel.py`**
    - **Purpose**: Maps ZIP codes to CBSA/MSA codes using the HUD crosswalk, selecting the highest-`BUS_RATIO` CBSA when a ZIP spans multiple metro areas.
    - **Output**: `cleaned_data/monthly_msa_panel_180mo.csv`.

11. **`02_cleaning_processing/build_etfs.py`**
    - **Purpose**: Aggregates the cleaned stock panel into five benchmark portfolios:
      `location_simple`, `location_weighted`, `industry_simple`, `industry_weighted`, and `market_weighted`.
    - **Output**: CSV files in `cleaned_data/etf_price/`.

12. **`02_cleaning_processing/add_benchmark_returns.py`**
    - **Purpose**: Constructs the leave-one-out regressors added back to each stock file:
      `market_return`, `industry_simple`, `industry_weighted`, and `location_return`.
      It also defines valid MSA-month cells using the current rule: at least 5 firms and at least 2 FF48 industries.
    - **Output**: Updated stock CSVs in `cleaned_data/stock_price/` and `cleaned_data/msa_validity_panel.csv`.

13. **`02_cleaning_processing/final_clean_for_regression.py`**
    - **Purpose**: Final pre-regression filter. It drops rows with missing regression inputs and keeps only stocks that have at least 24 clean months in at least one of the three five-year windows: 2008-2012, 2013-2017, and 2018-2022.
    - **Output**: Final analysis-ready stock CSVs in `cleaned_data/stock_price/`.

## Stage 3: Analysis
These scripts produce the summary statistics and the firm-level beta estimates reported in the draft paper.

14. **`03_analysis/table1_summary_stats.R`**
    - **Purpose**: Computes snapshot summary statistics for the filtered sample, including the number of firms per MSA and the number of industries per MSA in 2012, 2017, and 2022.
    - **Output**: `script/03_analysis/table1_panel_a.csv` and `script/03_analysis/table1_panel_b.csv`.

15. **`03_analysis/regress_betas.R`**
    - **Purpose**: Runs firm-level time-series regressions for Model 1:
      `ret ~ location_return + market_return + industry_simple`.
      It reports cross-sectional mean betas and t-statistics for all stocks and for the S&P 500 subsample.
    - **Output**: Terminal summary for the windows 2008-2012, 2013-2017, and 2018-2022.

16. **`03_analysis/regress_betas_model2_vw.R`**
    - **Purpose**: Runs firm-level time-series regressions for Model 2:
      `ret ~ location_return + market_return + industry_weighted`.
      It reports cross-sectional mean betas and t-statistics for all stocks and for the S&P 500 subsample.
    - **Output**: Terminal summary for the windows 2008-2012, 2013-2017, and 2018-2022.
